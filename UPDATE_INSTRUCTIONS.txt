LOVE QUILTS MANAGER — UPDATE 7.7.2
Ready-to-upload package

BEFORE UPLOADING
1. In the app, export a current backup.
2. Confirm both devices show all changes synced.
3. Download a backup ZIP of the current GitHub repository.

UPLOAD
1. Extract this ZIP.
2. Open the main/root folder of the GitHub repository.
3. Upload every extracted file and replace files with the same names.
4. Leave the firebase-test folder unchanged.
5. Wait for GitHub Pages to publish.
6. Completely close and reopen the app on each device.
7. Confirm the splash screen shows Update 7.7.2.

MARKING AN OLD NEED DISTRIBUTED
1. Open Needs and locate the need.
2. Choose Mark Distributed.
3. Confirm Quantity Distributed and Date Distributed.
4. Leave “Record as Love Quilts Out” UNCHECKED when the quilts were already removed from inventory.
5. Save Changes.

FUTURE DISTRIBUTIONS
- Check “Record as Love Quilts Out” only when the distribution has not already been entered on the Inventory screen.
- The app will create the matching Inventory Out transaction and prevent inventory from going below zero.

IMPORTANT
- Do not choose Create Real Shared Inventory again.
- Do not clear or reinitialize Firebase.
- Existing inventory, needs, names, approved accounts, and synchronization remain in place.
